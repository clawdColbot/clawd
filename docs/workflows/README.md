# Workflows Index

Colección de procesos estándar para consistencia y onboarding.

## Workflow List

| Workflow | Descripción | Tiempo | Frecuencia |
|----------|-------------|--------|------------|
| [shipyard_new_repo](shipyard_new_repo.md) | Crear repo GitHub + Shipyard | 10-15 min | On-demand |
| [responder_heartbeat](responder_heartbeat.md) | Procesar heartbeat de Clawdbot | 2-5 min | ~30 min |
| [crear_checkpoint](crear_checkpoint.md) | Capturar estado importante | 2-3 min | On-demand |
| [debug_problemas](debug_problemas.md) | Resolver problemas (curl repro) | Variable | On-demand |
| [escalacion](escalacion.md) | Decidir cuándo preguntar/actuar | 1 min | On-demand |

## Uso Rápido

```bash
# Crear nuevo repo
less ~/clawd/docs/workflows/shipyard_new_repo.md

# Responder heartbeat
less ~/clawd/docs/workflows/responder_heartbeat.md

# Debug
less ~/clawd/docs/workflows/debug_problemas.md
```

## Principios

1. **Un workflow por archivo:** Fácil de encontrar y mantener
2. **Checklist incluido:** No memorizar, seguir
3. **Ejemplos de comandos:** Copy-paste friendly
4. **Troubleshooting:** Soluciones comunes
